Easing Into Hugo Automation with Emacs: Part 1

Tue, Mar 22, 2016

After reading a post on Endless Parentheses about the author’s blogging workflow I instantly felt compelled to do something similar. Being an Emacs user, the thought of living inside of Emacs for as long as possible is always attractive. Of course, I could have simply used his configuration as he’s pointed to some of his configuration on GitHub’s, but his flow involves Jekyll, and I am using Hugo. Moreover, coming up with my own flow and possibly porting a package would be a good way for an elisp beginner to gain some knowledge.

So, here’s my first step at making Hugo play well with Emacs.

First, I wanted to be able to capture a post quickly. By using Org-Mode’s capture templates, this was pretty easy.

(Somewhere in my init.el)

(setq org-capture-templates
      '("p" "Post" plain (file (wm/hugo-new))
        (file "~/path-to-template-file")))

The wm/hugo-new function here just creates a file name with today’s date. This is what that function looks like:

(defun today-is ()
  (format-time-string "%Y-%m-%d"))

(defun wm/hugo-new ()
  (concat (file-name-as-directory "~/path-to-hugo-posts")
            (today-is) ".md"))

And the template that I am using is just the basic hugo template, with a custom function for formatting the date the way that Hugo likes it.

+++
Categories = []
Description = ""
Tags = []
date = "%(wm/insert-date-time)"
menu = "main"
title = ""
+++

%?

The wm/insert-date-time function:

(defun wm/insert-date-time ()
  "Insert current date-time string in full ISO 8601 format.
Example: 2010-11-29T23:23:35-08:00"
   (concat
    (format-time-string "%Y-%m-%dT%T")
    ((lambda (x) (concat (substring x 0 3) ":" (substring x 3 5)))
     (format-time-string "%z"))))

At this point, I can hit C-c c p and the screen splits vertically with a new markdown file created with the template ready to go. I just need to fill out the title, categories, and main content and hit C-c C-c to save the post and get back to any work that I was doing before I started the post.

Now that I have the file saved, I would like to be able to call the Hugo CLI to serve up the post I just created as well as build the posts to get them ready for publishing/pushing.

The base solution is eshell aliases.

In my eshell/alias file, I have 3 entries that help with Hugo.

alias blog-serve hugo -s="~/path-to-blog-src" server
alias blog-browse (browse-url "http://localhost:1313")
alias blog-publish hugo -s="~/path-to-blog-src"

blog-serve starts the server and makes the site available for browsing via the blog-browse alias. blog-publish is self-explanatory.

The capture, along with the functions and these aliases make it pretty easy for me to just whip up a post at the speed of thought, but there is still a little too much manual work here. The next part of this series I will experiment with porting one of the Jekyll packages for use with Hugo. It would be really cool to have an all in one solution for Hugo like there is for Jekyll.
