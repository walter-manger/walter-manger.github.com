Go Basics - Pointers

Fri, Apr 8, 2016

Pointers in Go are much more simpler to deal with than their C/C++ counterparts. For one, Go does not allow pointer arithmetic, which is a good thing. Pointer arithmetic can be really confusing at times, no matter how maticulous you write the code.

A simple example of a pointer in Go:

i := 99
p := &i

In this example, we are taking the address of i and assigning it to a variable p. If we were to print the result of p, we’d see something like 0x10434114. If you can imagine memory as a block of cells, i would be the name of one of the cells, p would be the cells’ identifier, and 99 would be the cells’ data.

The main advantage of using pointers is memory efficiency. Instead of copying values from one variable to another, the address is referenced. Work can still be done on the data that lives at that address through the dereferencing operator.

p = 100   // ERROR!
*p = 100  // Dereference :)

You can think of *p as i in this case. Consequently, the value of i is now 100.

Why would you use Pointers?

I think the more important question is where would you use pointers? I’ll dive into this more in later posts, but for now, know that Go makes use of a concept called Method Receivers, which define the way methods are attached to struct. If the struct needs to be modified by a method, a pointer for that struct is used.

Example:

type Blah struct {
  count int
}

func (b Blah) Hello() {
  // Not modifying the state of the struct
  // Blah is not a pointer
  fmt.Println("Hello")
  fmt.Println("Count: " + b.count)
}

func (b *Blah) Inc() {
  // Modifying state here
  // *Blah is a pointer, but no need to dereference
  b.count = b.count + 1
}

Don’t worry if this is confusing. I will be posting more about Structs, Pointers, and Methods. This is just a bite-sized introduction to pointers in Go.
